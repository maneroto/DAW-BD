MANE, TE PRESENTO A HTTP: SUS MÉTODOS DE PETICIÓN Y CÓDIGOS DE ERROR.

Inciemos explicándote el significado de HTTP

Mane no sabe el significado de HTTP, por lo que, nosotros, bien informados, le explicamos que el Hypertext Transfer Protocol (Protocolo de Transferencia de Hipertexto) es el set de reglas para la transferencia de archivos en la red. Así que cada que un usuario abre un navegador web, indirectamente está hacienddo uso de HTTP que a su vez corre sobre los protocolos de TCP/IP.
Y Mane también debería saber que HTTP incluye la idea de que los archivos pueden referenciar a otros archivos que al momento de ser seleccionados puedan brindar peticiones de transferencia adicionales. [1]

Mane, deberías saber que HTTP utiliza métodos de petición.

Los métodos de petición ayudan a que el usuario pueda realizar distintas peticiones a diversos archivos/páginas y que tanto el usuario, como la misma página, puedan interactuar con la información y otros archivos. Podemos encontrar diversas peticiones, algunas de ellas se muestran en la siguiente tabla. (Si puedes memorizar algunos estaría excelente).
Método Descripción
GET Transfiere una representación actual de el recurso seleccionado.
HEAD Igual que GET, pero solo transfiere la línea de estado y la sección del encabezado.
POST Realiza el procesamiento específico de recursos en la carga útil de solicitud.
PUT Reemplaza todas las representaciones actuales del recurso seleccionado con la carga útil requerida.
DELETE Remueve todas las representaciones actuales del recurso seleccionado.
CONNECT Establece un tunel con el servidor identificado por el recurso seleccionado.
OPTIONS Describe las opciones de comunicación para el recurso seleccionado.
TRACE Realiza una prueba de retorno de mensaje a lo largo de la ruta al recurso de destino.
[2]

También requieres saber que cada método de petición tiene sus pros y sus contras ya que por ejemplo, si quisieras mandar la información de usuario y contraseña a algún archivo, el método óptimo sería POST, ya que es un método que envía la información de manera segura, mientras que por ejemplo GET muestra dicha información en la misma URL y suele ser usado para que un navegador web acceda a una página de internet.[3]

Sí, Mane, HTTP también te responde y sí, hay errores.

HTTP tiene códigos de estado de respuesta que nos indican si se ha completado satisfactoriamente una solicitud HTTP específica. Las respuestas pueden ser de 5 clases:
  • Respuestas informativas
  • Respuestas satisfactorioas
  • Redirecciones
  • Errores de los clientes
  • Errores de los servidores
[4]
Los códigos de errores de los usuarios están comprendidos entre el código 400 y el código 451. Estos mensajes aparencen en diversas situaciones que implican al usuario, por ejemplo: sintáxis inválida (error 400), no se encuentra el contenido solicitado (error 404), no posee permisos necesarios (error 401 o 403), etc.

Los códigos de errores de los servidores están comprendidos entre el código 500 y el código 511. Estos mensajes aparecen cuando existe algún error de código o en los servidores que alojan a la página web, por ejemplo: error del servidor interno (error 500), ciclo detectado (error 508), autentificación de red requerida (error 511), etc.

Así que si en la página que planeas hacer aparece un código de error 404, tú tranquilo, todo esta 200 (o bueno, todo está bien) y no debes preocuparte por intentar corregirlo. Pero si ves un código de error 500 preocúpate, porque entonces ese fué un error tuyo que tienes que corregir.

Sobre el internet.

El internet se refiere al sistema de información global que está lógicamente conectado por una dirección única global basada en el IP, o subsecuentes conexiones. Es capáz de soportar comunicaciones a través del uso de TCP/IP y otros protocolos compatibles al IP.

Suele ser confundido con el WWW (World Wide Web), pero ésta es solo una convención que indica la dirección a la que pertenece la web y que ayuda a que el usuario pueda navegar a través de la web con un formato más amigable y fácil de entender.
[6]

Para crear una página web.

Se pueden crear páginas web haciendo uso de diversas herramientas y lenguajes, pero para ello debes conocer primero para qué te pueden ayudar cada una de ellas, por ejemplo:
  • Javascript: Javasacript permite hacer modificaciones a los tags, basándose en DOM, este lenguaje ayuda a determinar el comportamiento que una página web va a tener. Te proporciona a tí como programador, la opción de añadir diversos efectos e interacciones con la página que sin la ayuda de este lenguaje no podrías.

  • PHP: Es un lenguaje preprocesador que permite el ser combinado fácilmente con html y que al igal que javascript, permite el dar comportamientos a la página web.

  • XML: Te provee de diversos tags donde, por cada etiqueta debes de especificar sus atributos, la información que contiene y debe de tener una etiqueta de inicio y una de cierre.

  • XHTML: Que es la evolución de XML, lo cual es un intento por estandarizar y facilitar la escritura y lectura para la creación de las páginas web. Podría decirse que es el padre de HTML.

  • HTML5: Puede verse como un intérprete de texto a estructura web, ya que se encarga de que, al momento de escibirle los diversos tags y etiquetas que existen, genera esos elementos en la página. Su versión anterior fué HTML4 y sí, tienen varias diferencias. Parte de estas diferencias es que HTML5 ha despreciado o vuelto obsoletas varias etiquetas que existían en HTML4 (deprecated tags), por ejemplo: <center>, <frameset>, <big>, <b>, <i>, etc. A parte, en HTML5 no se requiere que se cieren diversas etiquetas que en HTML4 sí requieren su tag de cierre y, otra diferencia notable es que HTML5 añadió tags para que fuera más legible la creación de páginas como <section> y <article>
