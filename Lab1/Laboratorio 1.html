Inciemos explicándote el significado de HTTP

Mane no sabe el significado de HTTP, por lo que, nosotros, bien informados, le explicamos que el Hypertext Transfer Protocol (Protocolo de Transferencia de Hipertexto) es el set de reglas para la transferencia de archivos en la red. Así que cada que un usuario abre un navegador web, indirectamente está hacienddo uso de HTTP que a su vez corre sobre los protocolos de TCP/IP.
Y Mane también debería saber que HTTP incluye la idea de que los archivos pueden referenciar a otros archivos que al momento de ser seleccionados puedan brindar peticiones de transferencia adicionales. [1]

Mane, deberías saber que HTTP utiliza métodos de petición.

Los métodos de petición ayudan a que el usuario pueda realizar distintas peticiones a diversos archivos/páginas y que tanto el usuario, como la misma página, puedan interactuar con la información y otros archivos. Podemos encontrar diversas peticiones, algunas de ellas se muestran en la siguiente tabla. (Si puedes memorizar algunos estaría excelente).
Método Descripción
GET Transfiere una representación actual de el recurso seleccionado.
HEAD Igual que GET, pero solo transfiere la línea de estado y la sección del encabezado.
POST Realiza el procesamiento específico de recursos en la carga útil de solicitud.
PUT Reemplaza todas las representaciones actuales del recurso seleccionado con la carga útil requerida.
DELETE Remueve todas las representaciones actuales del recurso seleccionado.
CONNECT Establece un tunel con el servidor identificado por el recurso seleccionado.
OPTIONS Describe las opciones de comunicación para el recurso seleccionado.
TRACE Realiza una prueba de retorno de mensaje a lo largo de la ruta al recurso de destino.
[2]

También requieres saber que cada método de petición tiene sus pros y sus contras ya que por ejemplo, si quisieras mandar la información de usuario y contraseña a algún archivo, el método óptimo sería POST, ya que es un método que envía la información de manera segura, mientras que por ejemplo GET muestra dicha información en la misma URL y suele ser usado para que un navegador web acceda a una página de internet.[3]

Sí, Mane, HTTP también te responde y sí, hay errores.

HTTP tiene códigos de estado de respuesta que nos indican si se ha completado satisfactoriamente una solicitud HTTP específica. Las respuestas pueden ser de 5 clases:
  • Respuestas informativas
  • Respuestas satisfactorioas
  • Redirecciones
  • Errores de los clientes
  • Errores de los servidores

Los códigos de errores de los usuarios están comprendidos entre el código 400 y el código 451. Estos mensajes aparencen en diversas situaciones que implican al usuario, por ejemplo: sintáxis inválida (error 400), no se encuentra el contenido solicitado o se ha borrado (errores 404 y 410), no posee permisos necesarios (error 401 o 403), etc.
